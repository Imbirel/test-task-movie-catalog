# 3. Что такое ООП, и какие его ключевые принципы вы могли бы описать? При объяснении приводите примеры. Максимально подробно расскажите о том, как ООП реализовано в JavaScript.


# Ответ

ООП (Объектно-ориентированное программирование) — это методология программирования, основанная на представлении программы в виде совокупности объектов, каждый из которых является экземпляром определенного класса, а классы образуют иерархию наследования.

## Ключевые принципы ООП

1. Инкапсуляция — это объединение данных и методов для работы с ними внутри объекта, а также ограничение прямого доступа к внутренним деталям реализации.

Пример: У кофемашины есть кнопка «Сделать кофе». Вам не нужно знать, какое давление в бойлере или как работают клапаны; эти детали скрыты (инкапсулированы).

2. Наследование — механизм, позволяющий создать новый класс на основе существующего, перенимая его свойства и методы.

Пример: Класс «Транспорт» имеет метод двигаться. Класс «Автомобиль», наследуя «Транспорт», получает этот метод автоматически, но может добавить свои (например, открыть багажник).

3. Полиморфизм — возможность объектов с разным внутренним устройством иметь одинаковый интерфейс.

Пример: И у «Кошки», и у «Собаки» есть метод издатьЗвук(). Но кошка мяукнет, а собака гавкнет. Вы вызываете один и тот же метод, не заботясь о типе животного.

4. Абстракция — выделение только значимых характеристик объекта, игнорируя второстепенные детали.

Пример: При проектировании системы учета сотрудников для объекта «Человек» важны ФИО и должность, но не важны цвет глаз или группа крови.

## ООП в JavaScript

JavaScript долгое время был языком с прототипным наследованием, что отличает его от классических языков вроде Java или C++. С появлением стандарта ES6 (2015) добавился синтаксический сахар в виде классов, но под капотом всё равно остаются прототипы.

1. Классы и объекты

Класс в JS — это шаблон для создания объектов.

```js
class Animal {
  constructor(name) {
    this.name = name;
  }
  speak() {
    console.log(`${this.name} издает звук.`);
  }
}

const dog = new Animal("Бобик");
dog.speak(); // Бобик издает звук.
```

2. Инкапсуляция (Приватные поля)

В современном JS (стандарт 2022+) для реализации инкапсуляции используются приватные поля, начинающиеся с символа #.

```js
class BankAccount {
  #balance = 0; // Приватное поле

  deposit(amount) {
    this.#balance += amount;
  }

  getBalance() {
    return this.#balance;
  }
}

const account = new BankAccount();
account.deposit(100);
console.log(account.#balance); // Ошибка: поле приватное
```

3. Наследование

Реализуется с помощью ключевого слова extends и вызова super() для конструктора родителя.

```js
class Dog extends Animal {
  speak() {
    console.log(`${this.name} лает.`);
  }
}

const rex = new Dog("Рекс");
rex.speak(); // Рекс лает (переопределение/полиморфизм)
```

4. Прототипное наследование (Механика JS)

В JS у каждого объекта есть скрытое свойство [[Prototype]] (доступно через __proto__), которое ссылается на другой объект. Если мы ищем свойство или метод в объекте и не находим его, JS ищет его в прототипе.

```js
const animal = { eats: true };
const rabbit = { jumps: true };

rabbit.__proto__ = animal; // rabbit наследует от animal

console.log(rabbit.eats); // true
```

## Особенности реализации в JS:

- Функции — это объекты: В JS функции являются объектами первого класса.

- Динамическая природа: Можно добавлять методы к объектам «на лету» даже после их создания.

- Отсутствие интерфейсов: В чистом JS нет встроенного механизма interface (как в TypeScript), поэтому полиморфизм реализуется через «утиную типизацию» (если объект ведет себя как утка, значит это утка).
