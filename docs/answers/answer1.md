# 1. Как вы можете описать процесс распространения событий в JavaScript (Event Propagation)? Расскажите о его основных фазах. Какие есть способы применения знаний о фазах распространения событий на практике?


# Ответ

Распространение событий (Event Propagation) — это механизм, определяющий порядок, в котором элементы DOM получают уведомление о произошедшем событии.

## Фазы распространения событий.

Согласно стандарту DOM, процесс делится на три фазы:

1. Фаза погружения (Capturing phase): 
Событие спускается от корня документа `window` вниз по дереву DOM к целевому элементу. Обработчики, установленные на этой фазе, срабатывают первыми.

2. Фаза цели (Target phase): 
Событие достигает целевого элемента, на котором оно было инициировано.

3. Фаза всплытия (Bubbling phase): 
Событие поднимается вверх от целевого элемента к корню документа. Это поведение по умолчанию для большинства событий в JavaScript.

## Способы практического применения

Знание этих фаз позволяет эффективно решать следующие задачи:

- Делегирование событий: 
Вместо назначения обработчика каждому дочернему элементу (например, каждой строке в длинном списке), вы устанавливаете один обработчик на их общего родителя. Благодаря всплытию, событие от дочернего элемента поднимется к родителю, где его можно перехватить и обработать, проверив `event.target`. Это экономит память и упрощает работу с динамически добавляемыми элементами.

- Перехват событий на раннем этапе: 
Используя фазу погружения (аргумент `{ capture: true }` в `addEventListener`), можно поймать событие до того, как оно достигнет целевого элемента. Это полезно для создания глобальных фильтров, систем логирования или блокировки действий определенных компонентов.

- Управление потоком событий: 
С помощью метода `event.stopPropagation()` можно остановить дальнейшее распространение события (всплытие или погружение). Это часто используется в модальных окнах или выпадающих меню, чтобы клик внутри элемента не вызывал срабатывание обработчиков на внешних контейнерах.

- Предотвращение стандартного поведения: 
Метод `event.preventDefault()` позволяет отменить действие браузера по умолчанию (например, переход по ссылке или отправку формы), не останавливая при этом само распространение события по дереву.
