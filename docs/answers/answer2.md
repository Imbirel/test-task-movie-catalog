# Вопрос 2. 

### Расскажите, что такое Promise в JavaScript, как он работает, какие основные состояния у него бывают? Каким еще способомможно обработать асинхронный код? Какую роль в асинхронности играет событийный цикл (Event Loop)?


# Ответ

Promise в JavaScript — это объект, представляющий результат успешного или неудачного завершения асинхронной операции. Он служит контейнером для значения, которое станет известно позже.

## Как работает Promise и его состояния

Когда создается Promise, он запускает асинхронную задачу. Объект всегда находится в одном из трех состояний:

1. Pending (Ожидание): начальное состояние, операция еще не завершена.

2. Fulfilled (Исполнено): операция завершена успешно, возвращено значение.

3. Rejected (Отклонено): произошла ошибка, возвращена причина отказа.

Результат обрабатывается методами `.then()` (для успеха), `.catch()` (для ошибок) и `.finally()` (выполняется в любом случае).

## Другие способы обработки асинхронного кода

Помимо промисов, существуют следующие подходы:

- Callback-функции: передача функции в качестве аргумента, которая вызовется по завершении задачи. Использование вложенных колбэков часто приводит к «адскому дереву функций» (callback hell).

- Async/Await: современный «синтаксический сахар» над промисами. Позволяет писать асинхронный код, который выглядит и читается как синхронный, используя ключевые слова async перед функцией и await перед вызовом промиса.

## Роль Event Loop (Событийного цикла)

Event Loop — это механизм, который координирует выполнение кода, сбор событий и выполнение подзадач.

Его роль в асинхронности:

- Контроль стека вызовов: он проверяет, пуст ли основной стек выполнения.

- Очередь задач: когда асинхронная операция (например, таймер или запрос) завершается, ее результат попадает в очередь.

- Выполнение: если стек пуст, Event Loop берет первую задачу из очереди и переносит ее в стек для выполнения.

Для промисов существует специальная очередь микрозадач (microtasks), которая имеет приоритет и выполняется сразу после текущего кода, но до того, как браузер перейдет к отрисовке или обычным задачам (macrotasks).
